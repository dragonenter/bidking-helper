# 竞拍之王仓价值估算器 · 设计规格

**日期**：2026-05-05
**目标**：Windows 环境下运行的辅助工具，根据角色技能给出的信息（每轮陆续解锁），即时计算当前仓的期望价值、价值区间和出价建议。

---

## 1. 范围

### 1.1 本次交付（v1）

- 单文件 HTML，Windows 双击浏览器即开
- 7 个角色支持：维克托、艾哈迈德（石油哥）、拉文、伊森、伊莎贝拉、艾莎、老头
- 5 轮回合制流程：每轮按角色技能解锁可输入字段
- 输出：仓期望价值、95% 区间下限、当前轮出价建议（速胜倍率 / 保守 / 激进）

### 1.2 v1 不做（明确推迟）

- OCR 自动识别屏幕
- 桌面悬浮窗 / 托盘程序
- 单件物品级数据库（图鉴）
- 局后复盘
- 对手建模

## 3. 推理层（Inference Engine）

### 3.1 约束求解器（核心）

移植 VCY019/Bid-King-Calculator 的算法到 JS，支持以下约束类型：

- 件数：`count(quality) = X` 或 `count(quality) >= X`
- 总格数：`grid(quality) = X`
- 均格：`avg_grid(quality) = D`（关键：游戏只显示截断到 0.01 的均格，反推所有 (k, G) 满足 `⌈D·k⌉ < (D+0.01)·k`）
- 均价：`avg_price(quality) = P`（同样的截断处理）
- 总数：`total = N` 或 `total <= N`

输出：所有满足约束的 `(count_blue, count_purple, count_gold, count_red, count_grayscale)` 组合，及对应的总格数分布。

### 3.2 期望价值估算

对每个可行组合 c：

```
E[V | c] = c.purple × purple_item_avg
        + c.gold   × gold_item_avg
        + c.red    × red_item_avg_for_category
```

其中 `red_item_avg_for_category` 取决于用户在 UI 上选择的"仓品类"。如果未选品类，用 GLOBAL_EXACT 基准。

输出：
- **期望价值** = 所有可行组合的 E[V] 平均（按组合数等权或按格数加权）
- **保守下限** = 取所有组合 E[V] 的 5% 分位
- **乐观上限** = 95% 分位
- **可行组合数** = 越少表示信息越充分

### 3.3 出价建议

| 场景 | 建议出价 |
|---|---|
| R1-R4 速胜 | 期望价值 × 速胜倍率 + 安全边际 |
| R5 一槌定音 | 期望价值 × 0.95（略低于估值，避免买亏） |
| 信息不足（可行组合 > 100） | 提示玩家"此轮不建议出高价" |

```
┌─────────────────────────────────────────────────┐
│  竞拍之王 · 仓估值助手                          │
├──────────────────────┬──────────────────────────┤
│  左：输入面板         │  右：结果面板            │

│  [当前轮次 1/2/3/4/5] │                          │
│                      │  ━━ 价值区间 ━━━━━     │
│  ───────             │     [Y.YY ~ Z.ZZ]        │
│  R1 字段（按角色）     │                          │
│   总数 [___]         │  ━━ 出价建议 ━━━━━     │
│  R2 字段             │     R1 速胜: A.A         │
│   橙均格 [___]       │     R5 保守: B.B         │
│  ...                 │                          │
│                      │  ━━ 可行组合 ━━━━━     │
│  [计算] [清空]        │     当前还有 N 种可能     │
```

### 4.2 交互规则

- 角色切换 → 字段自动开关
- 输入新字段 → 自动重算
- 不满足约束 → 红字提示"无可行组合，检查输入"
- 状态保存到 localStorage（刷新不丢）

### 4.3 视觉

- 沿用 Oldzc 配色（紫/金/红 badge）
- 中文优先，紧凑布局，适配 1080P 屏幕

## 6. 验收标准

v1 必须通过的冒烟测试：

1. **基础 Demo**：维克托 R1 输入"总数=20"，仓品类=武器装备，能出"期望价值约 X、出价上限 Y"
2. **多约束收敛**：艾哈迈德 R1-R4 全输入后，可行组合数应 ≤ 5
3. **拉文终局**：R5 输入完整品质件数后，可行组合应锁定为 1
4. **错误输入兜底**：均格输入 0 或负数 → 友好错误提示，不白屏
5. **localStorage**：刷新页面后，上次输入的角色/品类/字段恢复
6. **离线运行**：断网双击打开 HTML，所有功能正常

测试由我（开发方）跑通后再交付。lilong 在 Windows 上验收时，至少跑场景 1+3。

## 8. 风险与已知不确定项

1. **角色技能字段映射可能不准**：sarkozyfan 给的映射是经验估计，需要 lilong 在游戏里验证
2. **品类价值数据未必精确**：sarkozyfan 数据来自社区统计，实际游戏内可能版本更新过
3. **均格反推算法的 maxCount 上限**：Oldzc 设的是 20，VCY019 没说，需要 lilong 确认仓内单一品质件数实际最大值
4. **白/绿/蓝是否完全可忽略**：v1 设为 0 是简化，如果实际有价值会偏低估
